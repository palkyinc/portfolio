Proyectos

Slam (2020-2022)

Ir al repositorio Github del proyecto

Descripción del problema:

Este es un proyecto que realicé como freelance sin otra participación, el objetivo es una aplicación web para la administración de un pequeño WISP. El principal problema a solucionar era poder gestionar y programar equipos Ubiquiti y Mikrotik de forma remota y automática, ya que se realizaba de manera manual, lo cual hacia la administración costosa en tiempo y con muchas posibilidades de error. Otros problemas adicionales a resolver fueron: registro de reclamos de clientes, incidencias y deudas técnicas globales; historial de tráfico de datos por cliente, historial de medición de estado de enlace de radio, monitoreo de enlaces a proveedores, inventario e historial de equipos. Otro desafío importante fue el hospedaje de la aplicación, ya que tiene que contactar a los equipos a programar de manera local.

Resolución:

Para el hosting decidí utilizar un servidor HP proliant con un Linux Alpine como sistema operativo. Con Docker generé dos contenedores, en uno con Apache, en el otro un servidor MySQL. La aplicación fue desarrollada principalmente con PHP-Laravel. Las vistas fueron trabajadas con Bootstrap, pero no me enfoque demasiado en el aspecto visual. Los gráficos de consumos de ancho de banda, son genrados con el complemento Chartisan. He diseña algunas vistas de formularios con Vue.js ya que deseaba que fuesen agiles e interactivas para el operador. Las vistas han sido filtradas por roles, y estos se asignan a usuarios. Los pedidos de información desde la aplicación hacia los equipos Ubiquiti se realizan a través de web services pero la programación se realiza con CLI enviados por SSH. En el caso de Mikrotik toda la comunicación se realiza por medio de la API que ofrecen estos dispositivos. La aplicación cuenta con una solución ante desastres debido a que el servidor no está en un cloud hosting. Diariamente se realiza un backup de la base de datos y los archivos que el sistema almacena, como imágenes y archivos de datos generados por los consumos de los clientes. El backup es comprimido y diariamente almacenado en la nube con una cuenta de google drive. También se realizó una vista donde se testea el estado del radioenlace de un cliente, esto se produce a demanda no automáticamente. El sistema genera email con reportes de incidentes pendiente, caídas de enlaces e informe de realización del backup. He utilizado el repositorio Git para la wiki de la aplicación utilizando Markdown.
